# 麻雀ビジョン - 仕様書

## 概要

あがった手牌の**役・翻数・符数・点数**を瞬時に計算・表示するWebアプリ。  
写真アップロードによる牌認識と、手動入力の両方に対応。  
PWAとして提供し、スマートフォンでも快適に操作できる。  
データの保存は一切行わず、その場の1回だけの計算に特化する。

## 採用ルール：雀魂（じゃんたま）ルール

### 四麻（4人麻雀）共通ルール

| 項目 | 設定 |
|------|------|
| 喰い断（食断） | あり |
| 後付け | あり |
| 赤ドラ | 各色1枚（5m・5p・5s） |
| 裏ドラ | あり（立直時） |
| カン裏ドラ | あり |
| 立直後の暗槓 | 待ちが変わらない場合のみ可 |
| ダブル役満 | あり |
| 数え役満 | 13翻以上で役満扱い |
| 役満の複合 | ダブル役満として計算 |
| 西入（西場突入） | なし（南4局で終了） |

### 三麻（3人麻雀）追加ルール

| 項目 | 設定 |
|------|------|
| 萬子 | 1・9のみ使用（2〜8は除外） |
| 北（4z） | 抜きドラとして扱う（1枚抜くたびに1翻・ドラ1加算） |
| プレイヤー | 東家・南家・西家の3人（北家なし） |
| 三麻での点数 | 3人で計算 |

## 機能要件

### 1. 手牌入力

#### 1-A. 写真アップロード入力

- スマートフォンのカメラ撮影 or ファイル選択で手牌画像をアップロード
- **サーバーサイド**のNext.js API Route（Edge runtime）がOpenAI Vision APIを呼び出し、牌を自動認識
- APIキーはサーバーの `.env` ファイルで管理（フロントエンドに露出しない）
- アクセス制御はCloudflare Accessで行う（Next.jsより手前でブロック）
- 認識精度の問題があるため、**必ず手動修正ができる**
- 認識失敗時は手動入力にフォールバック

**画像認識フロー：**
```
[ユーザーが写真を選択]
      ↓
[フロントエンドが /api/recognize に POST]
      ↓
[Next.js API Route (Edge runtime) が OpenAI Vision APIを呼び出し]
      ↓
[MPSZ形式の手牌文字列を返す]
      ↓
[フロントエンドで手牌に反映・修正可能]
```

**OpenAI Vision API へのプロンプト方針：**
- 手牌の牌を左から順に認識させる
- 出力形式を suit先頭MPSZ形式（例：`m234p567s234z11`）でJSON返却
- 認識できない牌は出力から除外し、confidence を下げる
- Vercel AI SDK の `generateObject` でスキーマ定義し型安全に取得
- 使用モデル：`gpt-4.1-mini`

#### 1-B. 手動タイル選択入力

- 画面に牌セレクターパレットを表示
- 牌をタップ/クリックして手牌に追加
- 追加した牌はタップで削除可能
- **ロン・ツモ共通：手牌13枚 ＋ 和了牌（ロン牌/ツモ牌）1枚を別指定**
  - 手牌13枚を選択後、14枚目が和了牌エリアに入る
  - ロン牌エリア/ツモ牌エリアのラベルはあがり方に応じて切り替わる

**入力できる牌の種類：**

| 種類 | 記号 | 三麻での扱い |
|------|------|-------------|
| 萬子 1, 9 | 1m, 9m | 使用可 |
| 萬子 2〜8 | 2m〜8m | **三麻では非表示** |
| 筒子 1〜9 | 1p〜9p | 使用可 |
| 索子 1〜9 | 1s〜9s | 使用可 |
| 東・南・西 | 1z〜3z | 使用可 |
| 北 | 4z | **三麻では「抜きドラ」として別管理（セレクターに非表示）** |
| 白・發・中 | 5z〜7z | 使用可 |
| 赤五萬 | 5mr | 使用可（三麻では萬子ごと除外） |
| 赤五筒 | 5pr | 使用可 |
| 赤五索 | 5sr | 使用可 |

**majiang-core の字牌番号：**
- 1z=東 / 2z=南 / 3z=西 / 4z=北 / 5z=白 / 6z=發 / 7z=中

**三麻での北牌の扱い：**
- 北（4z）は手牌には含めない
- 抜きドラ枚数を別途数値入力する（0〜4枚）
- 抜いた北はドラとして翻数に加算される

#### 2-A. 基本設定

| 設定項目 | 選択肢 | デフォルト |

#### 2-B. 特殊あがり条件

チェックボックス形式（複数選択可、矛盾する組み合わせは自動無効化）

| 条件 | 説明 | 制約 |
|------|------|------|
| 立直 | 立直宣言あり | |
| ダブル立直 | 第1ツモで立直 | 立直と排他 |
| 一発 | 立直/ダブル立直後1巡以内 | 立直またはダブル立直が必要 |
| 海底撈月（ハイテイ） | ツモ時の最後のツモ牌 | ツモ時のみ |
| 河底撈魚（ホーテイ） | ロン時の最後の捨て牌 | ロン時のみ |
| 嶺上開花 | カン後のツモ | ツモ時のみ |
| 槍槓（チャンカン） | 加槓牌でのロン | ロン時のみ |
| 天和 | 親の配牌であがり | 東家・ツモのみ |

| 項目 | 入力方法 | 備考 |
|------|----------|------|
| ドラ枚数 | ±ボタン（0〜） | 表示牌から自分で数えて入力 |

| 赤ドラ | 手牌入力時に赤五を選択 → 自動カウント | |

### 3. 計算・表示

`@kobalab/majiang-core` の `Majiang.Util.hule()` および `hule_mianzi()` を使用。

**majiang-core へのMPSZ文字列形式：**
- suit先頭形式：`m234p567s234z11`（スーツが先、数字が後）
- 赤ドラ：`0` で表現（例：赤5筒 = `p0`）
- 場風・自風パラメータ：0-indexed（東=0, 南=1, 西=2, 北=3）

#### 3-A. 役一覧

- 成立した全ての役を表示
- 各役の翻数を明示
- ドラ・赤ドラ・裏ドラ・抜きドラを区別して表示
- 役満時は「役満」「ダブル役満」等を強調表示

#### 3-B. 符数詳細表示（最重要機能）

符数の全内訳をビジュアルで表示する。各面子に実際の牌アイコンを添える。

**符の構成要素と計算ルール：**

| カテゴリ | 種別 | 符数 |
|----------|------|------|
| 副底（固定） | 常に | 20符 |
| 門前加符 | 門前ロン時 | +10符 |
| 自摸符 | ツモ時（平和以外） | +2符 |
| 順子（面子） | 全て | 0符 |
| 刻子・中張牌 | 明刻 | 2符 |
| 刻子・中張牌 | 暗刻 | 4符 |
| 刻子・幺九牌（1・9・字牌） | 明刻 | 4符 |
| 刻子・幺九牌 | 暗刻 | 8符 |
| 槓子・中張牌 | 明槓 | 8符 |
| 槓子・中張牌 | 暗槓 | 16符 |
| 槓子・幺九牌 | 明槓 | 16符 |
| 槓子・幺九牌 | 暗槓 | 32符 |
| 雀頭 | 一般牌 | 0符 |
| 雀頭 | 三元牌（白・發・中） | 2符 |
| 雀頭 | 場風牌 | 2符 |
| 雀頭 | 自風牌 | 2符 |
| 待ち・両面 | | 0符 |
| 待ち・辺張・嵌張・単騎 | | 2符 |
| 待ち・双碰 | | 0符 |

> **実装注意：** `hule()` は合計符数のみ返す。  
> 内訳の取得には `hule_mianzi()` で和了形一覧を取得し、  
> 各面子・雀頭・待ち形を独自解析して算出する。

#### 3-C. 点数表示

**表示項目：**

| 項目 | 例 |
|------|-----|
| 翻数・符数 | 「7翻 30符」 |
| 点数区分 | 「満貫」「跳満」「倍満」「三倍満」「役満」「ダブル役満」等 |
| あなたの得点 | 「+12,000点」 |
| 支払い内訳 | 誰がいくら払うか |

## インフラ・デプロイ

### Cloudflare Pages + Cloudflare Access

```
Internet
    ↓
Cloudflare Access（認証ゲート）
    ↓ 承認されたアカウントのみ通過
Cloudflare Pages（Next.js・GitHub連携自動デプロイ）
    ↓ /api/recognize など
Next.js API Route（Edge runtime）
    ↓
OpenAI Vision API（gpt-4.1-mini）
```

- **Cloudflare Access:** メールアドレス or Googleアカウントで許可リスト管理（50ユーザーまで無料）
- **APIキー管理:** Cloudflare Pagesの環境変数に `OPENAI_API_KEY` を設定
- **デプロイ:** GitHubのmainブランチへのpushで自動デプロイ

**Cloudflare Pagesダッシュボード設定：**

| 項目 | 設定値 |
|------|--------|
| Build command | `npx @cloudflare/next-on-pages` |
| Build output directory | `.vercel/output/static` |
| Node.js version | `20` |

## データフロー

```
[写真アップロード]                [手動牌選択]
      ↓                               ↓
[POST /api/recognize]         [手牌データ (MPSZ形式)]
      ↓
[OpenAI Vision API (Edge runtime)]
      ↓
[MPSZ文字列返却 → 手動修正可]
      ↓
            ┌──────────────────┐
            │  手牌データ確定   │
            │  (13枚 + 和了牌) │
            └──────────────────┘
                    ↓
            [ゲーム設定データ]
                    ↓
      [Majiang.Util.hule() + hule_mianzi()]
                    ↓
         [役・翻・符・点数データ]
                    ↓
              [画面表示]
```
